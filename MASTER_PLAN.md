## 3. 3 Etkileşim Modu

### 3.1 Mod 1 — Sesli Rehberlik Seansı

Kullanıcı direkt konuşur, hiçbir şey yazmak zorunda değil.

```
Kullanıcı Konuşur
    → Whisper (STT) — 0.5–1 saniye
    → GPT-4o (AI Rehber) — 1–2 saniye
    → Hume AI (Duygu Analizi) — paralel
    → ElevenLabs/Cartesia (TTS) — 0.5–1 saniye
    → Kullanıcı Duyar
    [Toplam latency: < 500ms]
```

**Özellikler:**
- Gerçek zamanlı, düşük gecikmeli konuşma (< 500ms)
- AI rehberin kişiselleştirilmiş sesi (klonlanmış, sıcak, samimi)
- Konuşma hızına ve tonuna dinamik uyum
- Sessizlik tespiti ve empati yönetimi
- Seans sonunda AI üretimi özet + öneriler

**Teknoloji:** VAPI.ai + OpenAI Whisper + GPT-4o + Hume AI + ElevenLabs

---

### 3.2 Mod 2 — Metin Tabanlı Seans

WhatsApp gibi sohbet arayüzüyle anlık yazışma.

```
Kullanıcı Yazar
    → GPT-4o (AI Rehber)
    → Metin Analizi + Duygu Çıkarımı
    → Protokol Yönlendirmesi
    → Yanıt (Streaming)
```

**Özellikler:**
- Anlık yanıt (streaming)
- Yazma göstergesi ("AI yazıyor" efekti)
- Uzun form günlük (journaling) modu
- Görsel araçlar: duygu seçim kartları, ölçekler, egzersizler
- Geçmiş seans hafızası (vector DB ile kişisel bağlam)

**Teknoloji:** OpenAI GPT-4o streaming + pgvector + Next.js

---

### 3.3 Mod 3 — Görüntülü AI Destekli Seans

AI rehber avatar karşısında oturularak konuşulur.

```
Kullanıcı Webcam Açar
    → Hume AI (Yüz Analizi — sadece duygu rehberliği için)
    → AI Avatar (gerçekçi yüz) + GPT-4o
    → Avatar Animasyon + Ses
    → Kullanıcı
```

**Özellikler:**
- Gerçekçi AI rehber avatarı (Tavus.io / HeyGen)
- Canlı yüz ifadesi analizi (Hume AI — duygu rehberliği)
- Göz teması simülasyonu
- Seans odası arkaplanı (sakin, huzurlu ortam)
- Tüm seans kaydı + transkript + özet

**Teknoloji:** Tavus.io / HeyGen + Hume AI + GPT-4o + WebRTC

> **Yasal Not:** Kamera verisi yalnızca anlık duygu rehberliği için işlenir. Ham video verisi saklanmaz. Kullanıcının kamerayı kapatma hakkı her zaman mevcuttur. Yüz analizi sonuçları tanı veya klinik değerlendirme olarak sunulamaz.

## 3.5 Avatar Sistemi — 4 Kategori (Kullanıcı Seçer)

### A — Fotorealistik İnsan (Simli.ai)
- **Kaynağı:** Gerçek insan fotoğrafından lipsync
- **Kütüphane:** 100+ lisanslı fotoğraf (çeşitli yaş/etnik/cinsiyet)
- **Kalite:** %95 gerçekçi
- **Maliyet:** $0.05-0.10/dk ($2-4 per 40dk session)
- **Hedef:** 25-45 yaş profesyoneller
- **UI:** Onboarding'de 10 profile göster → kullanıcı seçer

### B — UE5 MetaHuman (3D Gerçekçi)
- **Kaynağı:** Epic Games fotorealistik 3D karakter
- **Detay:** Tam vücut hareketleri, yüz animasyonu
- **Sunumu:** Pixel Streaming (150-300ms latency)
- **Kalite:** %90 gerçekçi, full-body
- **Maliyet:** $0.80/seans (GPU compute)
- **Hedef:** Premium UX odaklı kullanıcılar

### C — Anime/Pixel Sanat (2D Animasyon)
- **Kaynağı:** Flat design veya anime tarzı
- **Motor:** Spine 2D / Live2D
- **Render:** Canvas + WebGL (GPU gerektirmez)
- **Kalite:** Artistik, renkli, dinamik
- **Maliyet:** $0.01/seans
- **Hedef:** 16-24 yaş, gaming kültürü

### D — Çizgi Film Karakterler (Çocuklar) — YENİ
- **Karakterler:** 5+ Pixar tarzı: tavşan, peri, robot, astronot, denizaltı kaşifi
- **Kişilik:** Her karakter kendine özgü (cesur, meraklı, sakin, eğlenceli)

- **Maliyet:** $0.01/seans
- **Hedef:** 7-16 yaş (ebeveyn onayıyla), aile wellness

## 3.6 Video Kalite Sistemi — Auto + Manuel

### Otomatik Mod (Default)
```
Bağlantı Analizi Gerçek Zamanlı:
├─ WiFi 5G/6    → 4K 60fps (8-12 Mbps)  [Ultra]
├─ WiFi 5GHz    → 1080p 60fps (4-6)     [High]
├─ WiFi 2.4GHz  → 1080p 30fps (2-4)     [Medium]
├─ 4G LTE       → 720p 30fps (1-2)      [Standard]
├─ 3G           → 480p 24fps (0.5)      [Low]
└─ Zayıf Bağlantı → Ses Only            [Audio-only]
```

### Manuel Override (Ayarlar)
```
Video Kalitesi:
○ Otomatik (Önerilen)
○ 4K 60fps (Ultra — güçlü bağlantı lazım)
○ 1080p 60fps (Yüksek)
○ 1080p 30fps (Standart)
○ 720p 30fps (Ekonomik — az veri)
○ Ses Modu (görsel yok, sadece ses)
```

### Teknik Stack
- **Codec:** NVIDIA NVENC (server-side H.264/H.265)
- **Protokol:** HLS / DASH (adaptive bitrate)
- **Client:** HLS.js (web), ExoPlayer (Android), AVPlayer (iOS)
- **CDN:** CloudFront (20 bölge edge node)
- **Latency:** 100-350ms (WebRTC P2P + caching)

### Maliyet (40dk Seans)
| Kalite | Bandwidth | CDN Maliyet |
|--------|-----------|------------|
| 4K 60fps | 3.6 GB | $0.09 |
| 1080p 60fps | 1.8 GB | $0.045 |
| 720p 30fps | 0.9 GB | $0.023 |
| Ses Modu | 50 MB | $0.001 |

## 3.7 Apple Watch / Android Wear Entegrasyonu

### iOS — HealthKit API
```
Adım 1: Xcode → HealthKit capability
Adım 2: Info.plist → NSHealthShareUsageDescription
Adım 3: Onboarding → HKHealthStore().requestAuthorization()
  ├─ Uyku verisi (sleep)
  ├─ Kalp Ritmi Varyasyonu (HRV)
  └─ Adım sayısı (steps)
Adım 4: Background → HKObserverQuery → otomatik sync
```

### Android — Health Connect / Google Fit
```
Adım 1: build.gradle → health_connect dependency
Adım 2: AndroidManifest.xml → permissions
Adım 3: İzin isteği (Onboarding)
Adım 4: Background Worker → her gece 3:00'da sync
```

### Veri Akışı
```
Apple Watch / Wear OS
  ↓ (arka planda, kullanıcı hiç dokunmuyor)
Health App / Health Connect
  ↓ (HealthKit API, ~1dk sync)
Kaira React Native App
  ↓ (Supabase webhook)
health_data tablosu
  ↓
AI Analiz (GPT-4o)
  ↓
Sabah Bildirimi: "Dün 5.5 saat uyudun, bugün mood takip edelim"
```

### Web'de?
- **Hayır** — HealthKit = iOS native only
- **Çözüm:** Web kullanıcısı → manual input ("Dün kaç saat uyudun?")
- **Yönelim:** Mobil uygulamayı teşvik et (daha iyi veri)

### 6.8 Database Schema (PostgreSQL + pgvector)

```sql
-- USERS & AUTH
CREATE TABLE users (
  id UUID PRIMARY KEY,
  email VARCHAR UNIQUE NOT NULL,
  password_hash VARCHAR NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  last_login TIMESTAMP,
  deleted_at TIMESTAMP -- soft delete
);

-- PROFILES
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES users(id),
  first_name VARCHAR,
  age INT,
  country_code VARCHAR(2),
  timezone VARCHAR,
  language_pref VARCHAR,
  ai_voice_preference VARCHAR,
  avatar_preference VARCHAR,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP
);

-- SESSIONS
CREATE TABLE sessions (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  type VARCHAR (text|voice|video),
  duration_minutes INT,
  started_at TIMESTAMP,
  ended_at TIMESTAMP,
  rating INT (1-5),
  transcript TEXT,
  summary TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- MESSAGES
CREATE TABLE messages (
  id UUID PRIMARY KEY,
  session_id UUID REFERENCES sessions(id),
  sender VARCHAR (user|ai),
  content TEXT,
  embedding VECTOR(1536), -- pgvector for semantic search
  emotion_detected VARCHAR,
  created_at TIMESTAMP DEFAULT NOW()
);

-- MOOD LOGS
CREATE TABLE mood_logs (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  score INT (1-10),
  emotion VARCHAR,
  context TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- WELLNESS PLANS
CREATE TABLE wellness_plans (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  created_at TIMESTAMP,
  topics JSONB, -- week 1-7 topics
  current_week INT,
  status VARCHAR (active|completed|paused),
  updated_at TIMESTAMP
);

-- ASSESSMENTS (PHQ-9, GAD-7)
CREATE TABLE assessments (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  type VARCHAR (phq9|gad7),
  answers INT[], -- [0,1,2,3] array
  score INT,
  level VARCHAR (minimal|mild|moderate|severe),
  created_at TIMESTAMP
);

-- CRISIS EVENTS
CREATE TABLE crisis_events (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  risk_level VARCHAR (low|medium|high),
  trigger_content TEXT,
  actions_taken JSONB,
  hotline_offered VARCHAR,
  created_at TIMESTAMP
);

-- SESSION EMBEDDINGS (vector memory)
CREATE TABLE session_embeddings (
  id UUID PRIMARY KEY,
  session_id UUID REFERENCES sessions(id),
  embedding VECTOR(1536),
  context_snippet TEXT,
  created_at TIMESTAMP
);

-- CONSENT RECORDS (GDPR/KVKK compliance)
CREATE TABLE consent_records (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  consent_type VARCHAR (privacy|terms|research|marketing),
  granted BOOLEAN,
  timestamp TIMESTAMP,
  ip_address VARCHAR
);

-- DATA DELETION REQUESTS (GDPR Article 17)
CREATE TABLE data_deletion_requests (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  requested_at TIMESTAMP,
  completed_at TIMESTAMP,
  status VARCHAR (pending|processing|completed)
);

-- Indexes
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_sessions_user_id ON sessions(user_id);
CREATE INDEX idx_messages_session_id ON messages(session_id);
CREATE INDEX idx_mood_logs_user_id ON mood_logs(user_id);
CREATE INDEX idx_messages_embedding ON messages USING ivfflat (embedding vector_cosine_ops);

-- Row Level Security (RLS)
ALTER TABLE sessions ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can view own sessions" ON sessions
  FOR SELECT USING (auth.uid() = user_id);
```

### 7.4 Kriz Tespit Algoritması

```python
def detect_crisis(user_input: str, user_history: List[Message]) -> CrisisLevel:
    """
    Kriz seviyesi: low, medium, high
    """

    # 1. KEYWORD SCANNING (immediate)
    crisis_keywords = {
        "high": ["suicide", "kill myself", "end it", "no point living"],
        "medium": ["hurt myself", "harm", "overdose", "psychosis"],
        "low": ["hopeless", "worthless", "can't go on"]
    }

    # 2. SEMANTIC ANALYSIS (GPT-4o + Hume AI)
    semantic_risk = gpt4o.analyze_intent(
        input=user_input,
        context=user_history[-5:],
        prompt="Assess suicide/self-harm intent (0-1)"
    )

    # 3. BEHAVIORAL PATTERNS
    pattern_risk = analyze_trends(
        mood_logs[-30:],  # last 30 days
        session_frequency,
        crisis_event_history
    )

    # FINAL SCORE
    combined_score = (
        keyword_score * 0.4 +
        semantic_risk * 0.4 +
        pattern_risk * 0.2
    )

    if combined_score > 0.7:
        return CrisisLevel.HIGH
    elif combined_score > 0.4:
        return CrisisLevel.MEDIUM
    else:
        return CrisisLevel.LOW
```

```python
class MemoryOrchestrator:
    def store_session_memory(self, session: Session):
        """Her seansı vektör DB'ye kaydet"""

        # 1. Session özeti oluştur
        summary = gpt4o.summarize(
            messages=session.messages,
            max_tokens=200
        )

        # 2. Önemli noktaları çıkar
        key_points = {
            "topics": extract_topics(session.messages),
            "emotions": extract_emotions(session.messages),
            "triggers": extract_triggers(session.messages),
            "progress": extract_progress(session.messages),
            "techniques_used": extract_techniques(session.messages)
        }

        # 3. Embedding oluştur
        embedding = openai.Embedding.create(
            input=summary,
            model="text-embedding-3-small"
        )

        # 4. pgvector'e kaydet
        supabase.from_("session_embeddings").insert({
            "session_id": session.id,
            "embedding": embedding,
            "context_snippet": summary,
            "metadata": key_points
        })

    def retrieve_relevant_context(self, user_id: str, k=5):
        """Son 5 seansa benzer bağlamı bul"""

        # Current session intent
        current_intent_embedding = openai.Embedding.create(
            input=current_user_message,
            model="text-embedding-3-small"
        )

        # Semantic search
        similar = supabase.rpc(
            "match_documents",
            {
                "query_embedding": current_intent_embedding,
                "match_count": k,
                "user_filter": user_id
            }
        )

        return similar
```
